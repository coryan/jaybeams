1 ITCH-5.0 Messages {#itch50page}
2 =================
3 
4 Helper classes to parse ITCH-5.0 messages, as documented in the ITCH-5.0
5 protocol specification (hereafter 'the spec'):
6 
7 http://www.nasdaqtrader.com/content/technicalsupport/specifications/dataproducts/NQTVITCHspecification.pdf
8 
9 Each message type defined in the spec is represented by a class. The
10 directory also contains some common types, and classes that refactor
11 the headers common to all messages in the protocol.
12 
13 In general a message class meets the following constraints:
14 
15 @code
16 class foo_message {
17 public:
18  /**
19  * The message type identifier.
20  *
21  * ITCH-5.0 uses a single byte on the wire to identify the type of
22  * message. The identifier is an ASCII character, though in most
23  * platforms a character literal (for example 'S') will evaluate to
24  * the corresponding ASCII code (83 in the case of 'S'), this is not
25  * guaranteed. We use UTF-16 literals (in this example u'S')
26  * because these are available in C++11 and match ASCII for the
27  * characters used by the protocol (in C++14 we could use u8'S', but
28  * we are not using C++14 features).
29  */
30  static int const message_type = ...;
31 };

33 template<bool V>
34 class decoder<V,foo_message> {
35  /**
36  * Decode, and if V==true validate, an ITCH-5.0 buffer into the message type.
37  *
38  * Extract the field values from the buffer, and validate the fields
39  * against the protocol. The protocol makes many guarantees with
40  * respect to field values, for example, prices are only in the
41  * [0,200000] range, symbols and the locate codes are constant
42  * through the life of a session. This version of the decode
43  * function validates the fields, this is useful during testing to
44  * make sure our assumptions about the protocol are correct.
45  * However, in a production environment one might prefer to trust
46  * the protocol source and use unsafe_decode()
47  */
48  static foo_message r(std::size_t size, char const* buffer, std::size_t off);
49 };
