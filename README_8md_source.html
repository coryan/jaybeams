9 Another project to have fun coding.
10 
11 The JayBeams library performs relative time delay estimation of market
12 feeds. That is, given two feeds for the same inside quote data it
13 estimates, in near real-time which one is faster, and by how much.
14

16 - The installation instructions are in the INSTALL file.

18 ### Motivation
19 
20 The US equity markets have a lot of redundant feeds with basically the
21 same information, or where one feed is a super set of a second set.
22 When one has two feeds an interesting question is to know how much
23 faster is one feed vs. the other, or rather, how much faster it is
24 right now, because the latency changes over time. There are (usually)
25 no message IDs or any other simple way to match events in one feed
26 vs. events in the second feed. So the problem of "time-delay
27 estimation" requires some heavier computation, and doing this in
28 real-time is extra challenging. The current implementation is based
29 on cross-correlation of the two signals, using FFTs on GPUs to
30 efficiently implement the cross-correlations.
31 
32 You can find more details about the motivation, and the performance
33 requirements on [my posts](htts://coryan.github.io/)
34

40 #### So where is the code?

44 a challenge given the dependency on OpenCL libraries.

46 ### What is with the name?
47 
48 JayBeams is named after a [WWII electronic warfare](https://en.wikipedia.org/wiki/List_of_World_War_II_electronic_warfare_equipment) system.
49 The name was selected more or less at random from the Wikipedia list
50 of such systems, and is not meant to represent anything in
51 particular. It sounds cool, in a Flash Gordon kind of way.
