18  * Define a class used to represent prices in the ITCH-5.0 feed.
19  *
20  * The ITCH-5.0 protocol represent prices as fixed-point values. The
21  * number of decimal digits is specified in the field definition. For
22  * example, a Price(4) field represents prices with 4 decimal digits,
23  * i.e., of the 99999.9999 form. On the wire the number is actually
24  * stored as an integer. For example, the price 150.0100 would be
25  * stored on the wire as 1500100.
26  *
27  * @tparam wire_type an integer type, typically std::uint32_t for
28  * 4-byte wide fields, or std::uint64_t for 8-byte wide fields.
29  * @tparam denom the denominator for this field, for example Price(4)
30  * would use denom==10000, while Price(8) would use 100000000.
31  */
