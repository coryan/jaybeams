74 /**
75  * Wrap FFTW3 plan objects to automatically destroy them.
76  *
77  * The FFTW3 optimizes execution by pre-computing coefficients and
78  * execution plans for a DFT based on the original types, size and
79  * alingment of the data. These plans must be destroyed to release
80  * resources. FFTW3, being a C library, requires wrappers to automate
81  * the destruction of these objects.
82  *
83  * In addition, the FFTW3 library uses different names for the types
84  * that have single (fftwf_*), double (fftw_*) or quad-precision
85  * (fftwl_*). In C++ we prefer to hide such details in generics.
86  *
87  * @tparam in_timeseries_type the type of the input timeseries
88  * @tparam out_timeseries_type the type of the output timeseries
89  */
