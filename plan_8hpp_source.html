16 /**
17  * Wrap FFTW3 plan objects to automatically destroy them.
18  *
19  * The FFTW3 optimizes execution by pre-computing coefficients and
20  * execution plans for a DFT based on the original types, size and
21  * alingment of the data. These plans must be destroyed to release
22  * resources. FFTW3, being a C library, requires wrappers to automate
23  * the destruction of these objects.
24  *
25  * In addition, the FFTW3 library uses different names for the types
26  * that have single (fftwf_*), double (fftw_*) or quad-precision
27  * (fftwl_*). In C++ we prefer to hide such details in generics.
28  *
29  * @tparam in_timeseries_type the type of the input timeseries
30  * @tparam out_timeseries_type the type of the output timeseries
31  */
