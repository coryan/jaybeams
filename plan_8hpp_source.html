76 /**
77  * Wrap FFTW3 plan objects to automatically destroy them.
78  *
79  * The FFTW3 optimizes execution by pre-computing coefficients and
80  * execution plans for a DFT based on the original types, size and
81  * alingment of the data. These plans must be destroyed to release
82  * resources. FFTW3, being a C library, requires wrappers to automate
83  * the destruction of these objects.
84  *
85  * In addition, the FFTW3 library uses different names for the types
86  * that have single (fftwf_*), double (fftw_*) or quad-precision
87  * (fftwl_*). In C++ we prefer to hide such details in generics.
88  *
89  * @tparam in_timeseries_type the type of the input timeseries
90  * @tparam out_timeseries_type the type of the output timeseries
91  */
