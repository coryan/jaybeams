14 /**
15  * Wrap FFTW3 plan objects to automatically destroy them.
16  *
17  * The FFTW3 optimizes execution by pre-computing coefficients and
18  * execution plans for a DFT based on the original types, size and
19  * alingment of the data. These plans must be destroyed to release
20  * resources. FFTW3, being a C library, requires wrappers to automate
21  * the destruction of these objects.
22  *
23  * In addition, the FFTW3 library uses different names for the types
24  * that have single (fftwf_*), double (fftw_*) or quad-precision
25  * (fftwl_*). In C++ we prefer to hide such details in generics.
26  *
27  * @tparam precision_t the type of floating point value to use, must
28  * be either float, double or long double.
29  */
